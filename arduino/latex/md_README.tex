This folder contains the code to be uploaded to the Arduino. {\bfseries{Do not place code that should be run on the Jetson here!}}\hypertarget{md_README_autotoc_md1}{}\doxysection{Interfacing with the Arduino}\label{md_README_autotoc_md1}
The {\ttfamily arduino-\/cli} tool is needed to control upload to the Arduino from the commandline, and will be used to soft reset the Arduino when required.\hypertarget{md_README_autotoc_md2}{}\doxysection{Installing $<$tt$>$arduino-\/cli$<$/tt$>$}\label{md_README_autotoc_md2}
The {\ttfamily arduino-\/cli} binary file is contained in the {\ttfamily docker} folder in the root of this project, and should be installed automatically when you start the Docker instance. Note that there are {\bfseries{two}} versions present in the folder, one for x86 architecture and one for ARM architecture. The ARM one is used by default as the Jetson runs on ARM architecture. {\bfseries{If your system uses x86}}, be sure to rename the {\ttfamily arduino-\/cli\+\_\+x86} binary to {\ttfamily arduino-\/cli} so that the right version is installed.

If you are not using the Docker instance, you may alternatively refer to the \href{https://arduino.github.io/arduino-cli/latest/installation/}{\texttt{ installation steps}} to install it. Remember to add the downloaded binary to your \href{https://refspecs.linuxfoundation.org/FHS_3.0/fhs/ch03s04.html}{\texttt{ {\ttfamily /bin} path}}!

After installation, install the avr core and code dependencies\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{arduino-\/cli core install arduino:avr}
\DoxyCodeLine{arduino-\/cli lib install SimpleSerialProtocol}
\DoxyCodeLine{arduino-\/cli lib install Servo}

\end{DoxyCode}


Compile the code. The header files in this directory may be included with the {\ttfamily -\/-\/library} option\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{arduino-\/cli compile -\/-\/fqbn arduino:avr:nano:cpu=atmega328old -\/-\/library . main/main.ino}

\end{DoxyCode}


And upload it\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\# You may need to replace /dev/ttyUSB0 with the right serial port}
\DoxyCodeLine{arduino-\/cli upload -\/p /dev/arduino0 -\/-\/fqbn arduino:avr:nano:cpu=atmega328old main/main.ino}

\end{DoxyCode}
\hypertarget{md_README_autotoc_md3}{}\doxysection{$<$tt$>$udev$<$/tt$>$ rules}\label{md_README_autotoc_md3}
To make life easier, {\ttfamily udev} rules are used to assign fixed device nodes to the Arduino and IMU. As a result, these device names are hardcoded in the code.

{\ttfamily /etc/udev/rules.d/99-\/arduino.\+rules}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\# Detect arduino \& assign fixed device node.}
\DoxyCodeLine{ACTION=="{}add"{}, ATTRS\{idVendor\}=="{}1a86"{}, ATTRS\{idProduct\}=="{}7523"{}, MODE="{}0666"{}, GROUP="{}dialout"{}, SYMLINK+="{}arduino0"{}}

\end{DoxyCode}


{\ttfamily /etc/udev/rules.d/99-\/imu.\+rules}\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{\# Detect imu \& assign fixed device node.}
\DoxyCodeLine{ACTION=="{}add"{}, ATTRS\{idVendor\}=="{}0403"{}, ATTRS\{idProduct\}=="{}6001"{}, MODE="{}0666"{}, GROUP="{}dialout"{}, SYMLINK+="{}imu0"{}}

\end{DoxyCode}
